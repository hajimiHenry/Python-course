# args 和 kwargs 讲义

---

## 这份讲义要解决什么问题

很多同学学到这里会把下面几件事混在一起：

- `a=1, b=2` 这种普通关键字传参
- `*args` 这种可变位置参数
- `**kwargs` 这种可变关键字参数
- `def foo(*, x, y)` 里那个单独的 `*`
- `func(*lst)` / `func(**d)` 这种调用时的解包

它们都长得像星号，所以特别容易看晕。

这份讲义的目标只有一个：

> 把这些“长得像一家人”的写法彻底分开。

---

## 一句话先建立全局观

你可以先死记这三句，后面再慢慢理解：

- `*args`：收集多出来的“位置参数”
- `**kwargs`：收集多出来的“关键字参数”
- `name=值`：这只是普通的关键字传参，不等于 `**kwargs`

如果你先把这三句记住，后面基本就不会完全迷路。

---

## 先分清什么叫“位置参数”和“关键字参数”

看一个最普通的函数：

```python
def introduce(name, age):
    print(name, age)
```

调用方式 1：

```python
introduce('张三', 18)
```

这里是按位置传参：

- 第一个值给 `name`
- 第二个值给 `age`

调用方式 2：

```python
introduce(name='张三', age=18)
```

这里是按参数名传参，也叫关键字传参。

所以你先记住：

- “位置参数”说的是按顺序传
- “关键字参数”说的是按名字传

这时候还没有 `*args` 和 `**kwargs` 出场。

---

## 第一部分：普通关键字传参，不等于 kwargs

这是最容易混淆的地方。

比如：

```python
def introduce(name, age):
    print(name, age)


introduce(name='张三', age=18)
```

这里虽然用了“关键字”的方式传参，但这还不是 `kwargs`。

为什么？

因为这里的参数个数是固定的：

- `name` 就是 `name`
- `age` 就是 `age`

函数在定义的时候，已经提前写死了它们。

所以：

- `introduce(name='张三', age=18)` 是普通关键字传参
- `**kwargs` 是“关键字参数的数量都不确定，先全收进来再说”

这两个不是一回事。

---

## 第二部分：什么是 `*args`

### 1. 它是干什么的

`*args` 用来接收“数量不确定的位置参数”。

先看例子：

```python
def add(*args):
    print(args)
```

调用：

```python
add()
add(10)
add(10, 20, 30)
```

你可以把它理解成：

- `add()` 时，`args` 是空元组 `()`
- `add(10)` 时，`args` 是 `(10,)`
- `add(10, 20, 30)` 时，`args` 是 `(10, 20, 30)`

所以 `*args` 的本质是：

> 把多出来的位置参数，打包成一个元组。

### 2. 为什么是元组

因为位置参数最重要的是“顺序”。

元组天然就是有顺序的，所以 Python 用元组来装它们，非常合理。

### 3. `args` 这个名字可以改吗

可以。

比如：

```python
def add(*numbers):
    print(numbers)
```

这和 `*args` 在语法上完全一样。

只是大家约定俗成常写成 `args`，就像打招呼都说“你好”一样，不是法律规定，但最好别乱改得太离谱。

---

## 第三部分：什么是 `**kwargs`

### 1. 它是干什么的

`**kwargs` 用来接收“数量不确定的关键字参数”。

看例子：

```python
def show_info(**kwargs):
    print(kwargs)
```

调用：

```python
show_info()
show_info(name='张三')
show_info(name='张三', age=18, city='北京')
```

你可以理解成：

- `show_info()` 时，`kwargs` 是空字典 `{}`
- `show_info(name='张三')` 时，`kwargs` 是 `{'name': '张三'}`
- 传多个关键字参数时，会变成一个字典

所以 `**kwargs` 的本质是：

> 把多出来的关键字参数，打包成一个字典。

### 2. 为什么是字典

因为关键字参数的核心不是顺序，而是“参数名: 参数值”的对应关系。

字典正好就是这种结构。

### 3. `kwargs` 这个名字可以改吗

也可以。

比如：

```python
def show_info(**data):
    print(data)
```

语法完全合法。

但同样建议你遵守惯例，优先写 `kwargs`，因为别人一看就知道你要收集关键字参数。

---

## 第四部分：最容易混的三兄弟

请你认真看下面这三个例子，它们长得像，意思完全不同。

### 1. 普通关键字传参

```python
def f(name, age):
    print(name, age)


f(name='张三', age=18)
```

含义：

- 参数名和数量早就定死了
- 只是调用时不用位置，而是用名字来传

### 2. `*args`

```python
def f(*args):
    print(args)


f(1, 2, 3)
```

含义：

- 参数数量不确定
- 传进来的多个位置参数被收集到元组里

### 3. `**kwargs`

```python
def f(**kwargs):
    print(kwargs)


f(name='张三', age=18)
```

含义：

- 参数数量不确定
- 传进来的多个关键字参数被收集到字典里

你可以这样粗暴记忆：

- 固定参数名，用 `name=值` 就是普通关键字传参
- 不固定参数个数，收位置参数，用 `*args`
- 不固定参数个数，收关键字参数，用 `**kwargs`

---

## 第五部分：函数定义时，参数是怎么分配的

看这个函数：

```python
def demo(a, b, *args, **kwargs):
    print('a =', a)
    print('b =', b)
    print('args =', args)
    print('kwargs =', kwargs)
```

如果这样调用：

```python
demo(10, 20, 30, 40, x=1, y=2)
```

那么参数分配过程是：

- `a = 10`
- `b = 20`
- 多出来的位置参数 `30, 40` 进入 `args`
- 多出来的关键字参数 `x=1, y=2` 进入 `kwargs`

所以结果大概是：

```python
a = 10
b = 20
args = (30, 40)
kwargs = {'x': 1, 'y': 2}
```

你把这个过程想成“前台分流”就行：

- 固定座位的，先安排到固定参数
- 剩下散客里，按位置来的塞进 `args`
- 带名字来的塞进 `kwargs`

---

## 第六部分：为什么装饰器里老写 `*args, **kwargs`

这是你现在最该彻底看明白的地方。

比如：

```python
def record_time(func):
    def wrapper(*args, **kwargs):
        result = func(*args, **kwargs)
        return result
    return wrapper
```

为什么这里要写成这样？

因为装饰器里的 `wrapper` 要去替代原函数。

但问题是：

- 原函数可能有 1 个参数
- 也可能有 3 个参数
- 也可能既有位置参数，又有关键字参数

那 `wrapper` 怎么提前知道原函数长什么样？

它不知道。

所以最稳的办法就是：

- 位置参数我全收下，放进 `args`
- 关键字参数我全收下，放进 `kwargs`
- 然后原封不动再交给原函数

这就是：

```python
func(*args, **kwargs)
```

所以你现在应该明白了：

> 装饰器里写 `*args, **kwargs`，本质是在说：“不管你传什么参数来，我先全接住，再转交给原函数。”

---

## 第七部分：单独一个 `*` 又是什么

这个也很容易和 `*args` 混。

看例子：

```python
def greet(*, name, age):
    print(name, age)
```

它的意思不是“收集参数”。

这里单独一个 `*` 的作用是：

> 从这个位置往后，后面的参数必须使用关键字传参。

所以：

```python
greet(name='张三', age=18)   # 正确
greet('张三', 18)            # 错误
```

你可以把它理解为一个“分界线”：

- `*` 前面可以按位置传
- `*` 后面必须按名字传

这和 `*args` 完全不是一回事。

---

## 第八部分：定义时的星号，和调用时的星号，不是一回事

这是第二个大坑。

### 1. 定义函数时

```python
def f(*args, **kwargs):
    pass
```

这里的意思是“接收参数”。

- `*args` 接收多个位置参数
- `**kwargs` 接收多个关键字参数

### 2. 调用函数时

```python
nums = [10, 20, 30]
info = {'name': '张三', 'age': 18}
```

调用：

```python
f(*nums)
f(**info)
```

这里的意思是“拆开参数”。

- `*nums` 是把列表拆成位置参数
- `**info` 是把字典拆成关键字参数

所以一定要分清：

- 定义时：星号是在“收”
- 调用时：星号是在“拆”

这个思路一旦明白，很多代码都会瞬间顺眼。

---

## 第九部分：参数顺序规则

你不需要一上来就全背下来，但最好眼熟。

一个比较完整的参数顺序通常是：

```python
def func(a, b, /, c, d=10, *args, e, f=20, **kwargs):
    pass
```

从左到右大致是：

- 强制位置参数
- 普通参数
- 默认值参数
- `*args`
- 命名关键字参数
- `**kwargs`

对你现阶段来说，先记住最常见的一版就够了：

```python
def func(a, b, *args, **kwargs):
    pass
```

别一上来就把自己背进医院，没必要。

## 第十部分：常见错误和误区

### 误区 1：把普通关键字传参当成 `kwargs`

错误理解：

```python
f(name='张三')
```

就一定是 `kwargs`。

不对。

只有函数定义里真的写了 `**kwargs`，多出来的关键字参数才会被收集成字典。

### 误区 2：把 `*` 和 `*args` 当成一回事

不对。

- 单独的 `*` 是“后面参数必须写参数名”
- `*args` 是“收集多余的位置参数”

### 误区 3：把 `**kwargs` 写成 `**args`

语法上不是不行，但非常不推荐。

比如：

```python
def foo(**args):
    print(args)
```

这段代码能运行，但名字很别扭。

因为：

- `args` 一般默认表示位置参数
- `kwargs` 一般默认表示关键字参数

能不反着命名，就别反着来折磨未来的自己。

### 误区 4：以为 `args` 和 `kwargs` 是关键字

不是。

真正有语法意义的是：

- `*`
- `**`

`args` 和 `kwargs` 只是约定俗成的变量名。

---

## 第十一部分：什么时候该用，什么时候别乱用

### 适合用 `*args` / `**kwargs` 的场景

- 你真的不确定会传多少参数
- 你在写通用函数
- 你在写装饰器
- 你在写对外接口，希望更灵活

### 不太适合滥用的场景

- 参数其实很固定
- 你明明知道就 2 个参数，还硬写成 `*args`
- 为了“显得高级”把普通函数写得很绕

记住一句很重要的话：

> 灵活不是目的，可读性才是底线。

如果参数很明确，老老实实写清楚函数签名，通常更好。

---

## 第十二部分：一个最实用的记忆法

把函数调用想成“寄快递”：

- 按顺序直接丢过去的包裹，是位置参数
- 带标签写明“给谁”的包裹，是关键字参数
- `*args` 是收一堆没写名字、按顺序来的包裹
- `**kwargs` 是收一堆写了名字标签的包裹

再记一遍：

- `*args` -> 元组 -> 位置参数
- `**kwargs` -> 字典 -> 关键字参数

这两句最好背到条件反射。

---

## 最后做个总结

今天真正要带走的只有 6 条：

1. `name='张三'` 这种写法只是普通关键字传参，不等于 `**kwargs`
2. `*args` 收集多出来的位置参数，结果是元组
3. `**kwargs` 收集多出来的关键字参数，结果是字典
4. 单独的 `*` 是分界线，表示后面的参数必须用关键字传
5. 定义函数时的 `*` / `**` 是“收集参数”
6. 调用函数时的 `*` / `**` 是“拆开参数”

如果这 6 条你已经真的懂了，那这部分你就不是“会背”，而是真的会用了。

## 二、IPO：写程序时脑子里最该有的骨架

`IPO` 是：

- `Input`：输入
- `Process`：处理
- `Output`：输出

任何一个程序，本质上都能往这三个部分上靠。

例如：判断一个数是不是水仙花数。

- 输入：用户输入一个整数
- 处理：把每一位拆出来，做幂运算求和，再比较
- 输出：打印“是”或“不是”

你现在写题时，已经在用 IPO 了，只是还没有把它当作一种明确的方法。

### 为什么 IPO 重要

因为很多初学者一写题就容易糊成一坨：

- `input()` 写一点
- 计算写一点
- `print()` 再穿插一点

结果程序能跑，但结构很乱。

如果你先按 IPO 拆，就会清楚很多：

1. 输入到底是什么
2. 真正的计算逻辑是什么
3. 输出只是展示，不应该和计算搅在一起

### 你以后做题时的固定问法

每次看到题目，先问自己三句：

1. 输入是什么？
2. 处理逻辑是什么？
3. 输出要长什么样？

你如果能养成这个习惯，代码质量会立刻上一个台阶。

## 三、算法和流程图：不是画图炫技，是为了理清逻辑

### 什么是算法

算法就是：

**为了解决一个具体问题，设计出来的一组有限步骤。**

关键词有两个：

- 有限
- 有顺序

比如“求 1 到 n 的和”就是算法问题：

1. 读入 `n`
2. 准备一个变量 `total = 0`
3. 让数字从 1 加到 `n`
4. 输出 `total`

### 流程图的意义

流程图不是必须会画得很漂亮，而是让你理解：

- 顺序结构
- 分支结构
- 循环结构

这三种结构，几乎构成了你当前所有题目的骨架。

### 对你当前最有用的理解

你现在不用专门去学复杂流程图符号标准，但必须心里有这个意识：

- 先做什么
- 条件怎么判断
- 哪一段会重复
- 什么时候结束

如果脑子里这条线不清楚，代码就会写得东一榔头西一棒子。

## 四、编译和解释：为什么 Python 运行方式和 C 不一样

PPT 里有一块你现在仓库里几乎没系统讲过，就是：

- 编译
- 解释

### 编译是什么

编译是：

**先把源代码整体翻译成机器能直接执行的目标代码，再运行。**

像 C、C++ 更接近这种风格。

特点：

- 运行前先生成可执行结果
- 运行通常更快
- 改完代码一般要重新编译

### 解释是什么

解释是：

**程序运行时，一边翻译，一边执行。**

Python 更接近这种风格。

特点：

- 开发更灵活
- 改完代码直接运行
- 更适合教学、脚本、快速开发

### 这对你有什么实际意义

你现在最该记住的不是教科书定义，而是：

- Python 很适合你当前学习阶段
- 因为它反馈快，试错成本低
- 你写一点就能跑一点

这就是为什么你现在适合多练小程序，而不是只盯着理论。

## 五、Python 语法的“元规则”：不是知识点主角，但非常关键

这部分你平时在用，但没有系统整理过。

## 5.1 缩进

Python 用缩进表达代码层级。

这件事特别重要，因为在很多语言里，大括号决定代码块；在 Python 里，**缩进就是结构本身的一部分**。

例如：

- `if` 下面缩进去的代码，表示条件成立时执行
- `for` 下面缩进去的代码，表示循环体
- 函数定义后面缩进去的代码，表示函数体

### 你要形成的习惯

- 不把缩进当成“排版”
- 要把缩进当成“语法”

你那个未完成的 `code/day16_1.py` 就属于很典型的提醒：  
函数定义后，如果没有缩进代码块，语法就直接不成立。

## 5.2 注释

注释是写给人看的，不是写给 Python 执行的。

作用：

- 解释这段代码想干什么
- 说明容易误解的地方
- 留下思路，而不是重复代码字面意思

### 什么是好注释

好注释：

- 解释“为什么这样做”
- 提醒边界条件
- 说明复杂逻辑

坏注释：

- `x = x + 1  # 给x加1`

这种没有信息增量。

## 5.3 命名规则

变量命名本质上就是给数据起名字。

合法命名的基本规则：

- 可以有字母、数字、下划线
- 不能以数字开头
- 区分大小写
- 不能和保留字重名

### 初学阶段最推荐的命名习惯

- 变量名尽量见名知意
- 不要总写 `a`、`b`、`c`
- 除了临时循环变量，尽量让名字表达用途

例如：

- `total`
- `score`
- `student_name`
- `is_prime`

这样的名字比 `x1`、`abc` 强太多。

## 5.4 保留字

保留字就是 Python 自己占用的单词，你不能再拿来当变量名。

比如：

- `if`
- `else`
- `for`
- `while`
- `def`
- `class`
- `import`
- `try`

### 为什么要知道这个

不是让你背所有关键字，而是要知道：

- 写变量名时如果报语法错误
- 有时候不是逻辑错了
- 是你不小心碰了保留字

## 六、库导入：你以后会天天用，但现在还没系统总结

这部分在 PPT 里讲得很基础，但对你很实用。

Python 里常见的导入方式有 3 种。

## 6.1 `import 模块名`

```python
import time
```

调用时写：

```python
time.sleep(1)
```

优点：

- 清楚
- 不容易重名
- 一眼知道函数来自哪个模块

这是最稳妥、最推荐的方式。

## 6.2 `from 模块名 import 名字`

```python
from datetime import datetime
```

调用时可以直接写：

```python
now = datetime.now()
```

优点：

- 写起来短

缺点：

- 如果导入很多同名函数，容易混淆来源

## 6.3 `import 模块名 as 别名`

```python
import random as rd
```

调用时：

```python
rd.randint(1, 10)
```

常见用途：

- 模块名太长
- 想写得更短

### 你现在最该采用的习惯

初学阶段优先使用：

- `import 模块名`
- 或 `from 模块名 import 一个非常明确的对象`

尽量少用：

- `from xxx import *`

因为它会把命名空间弄得很乱。

## 七、数字类型里你还没系统学透的部分

## 7.1 进制

整数不只有十进制。

Python 里常见写法：

- 十进制：`10`
- 二进制：`0b1010`
- 八进制：`0o12`
- 十六进制：`0xA`

它们本质上只是**同一个数的不同写法**。

例如：

- `0b1010`
- `0o12`
- `0xA`
- `10`

都表示十进制的 `10`。

### 你什么时候会用到

现在做题不常用，但以后会在这些场景碰到：

- 权限位
- 位运算
- 编码
- 调试底层数据

## 7.2 浮点数误差

这个点特别值得你早点知道。

很多初学者以为：

```python
0.1 + 0.2 == 0.3
```

一定是 `True`。

实际上在很多语言里，这种计算可能出现精度误差。

原因不是 Python “算错了”，而是：

- 计算机内部用二进制近似表示小数
- 某些十进制小数不能被二进制精确表示

### 正确态度

以后遇到浮点数比较：

- 不要太迷信“绝对相等”
- 要理解“近似表示”的概念

## 7.3 科学计数法

Python 里可以直接写：

```python
4.3e-3
9.6E5
```

它们分别表示：

- `4.3 × 10^-3`
- `9.6 × 10^5`

这个在数据处理、科学计算、很大或很小的数里非常常见。

## 7.4 复数

这是你当前学习里基本没真正用过的一块。

Python 支持复数，写法像这样：

```python
z = 3 + 4j
```

注意：

- Python 用 `j` 表示虚数单位
- 不是数学课上常写的 `i`

可以这样取：

```python
z.real   # 实部
z.imag   # 虚部
```

### 你现在需要掌握到什么程度

不用深挖复数运算，只要知道：

- Python 有这个类型
- 它是什么
- 基本长什么样
- 有 `.real` 和 `.imag`

就够了。

## 八、字符串里你还该补的底层点

你已经学过很多字符串方法了，但下面这几块值得系统梳理。

## 8.1 转义字符

当你想在字符串里表达一些特殊效果时，会用到转义字符。

常见的有：

- `\n`：换行
- `\t`：制表符
- `\\`：反斜杠本身
- `\"`：双引号
- `\'`：单引号
- `\r`：回车

### 为什么有用

比如做文本进度条时，`\r` 很关键。

它的作用不是换行，而是：

**把光标拉回当前行开头。**

这样你后面再打印，就能覆盖这一行，形成“动态刷新”的效果。

## 8.2 Unicode、`chr()`、`ord()`

你已经零散见过，但还没真正把这条线串起来。

### Unicode 是什么

Unicode 可以理解成：

**给全世界字符统一编号的一套标准。**

比如：

- `'A'`
- `'中'`
- `'🐍'`

背后都对应一个数字编码。

### `ord()`

`ord(字符)`：字符 -> 编码

### `chr()`

`chr(编码)`：编码 -> 字符

例如：

```python
ord('A')
chr(65)
```

### 这对你有什么意义

它能帮你理解：

- 字符本质上也可以看作数字
- 字符串处理并不只是“看起来像文字”
- 编码问题、乱码问题背后都和这个有关

## 九、`time` 库：你现在特别值得补的一块

这部分在你当前仓库里几乎没有系统覆盖，但实际很常用。

## 9.1 `sleep()`

```python
import time
time.sleep(1)
```

意思是：

程序暂停 1 秒。

你已经在自己的代码里用过类似思路，但还没有系统理解 `time` 模块本身。

## 9.2 `time.time()`

返回当前时间戳，一个浮点数。

你可以粗略理解成：

**从某个固定起点到现在经过了多少秒。**

## 9.3 `ctime()`

把当前时间变成人更容易看懂的字符串。

## 9.4 `strftime()`

把时间按你想要的格式输出。

比如：

- 年月日
- 时分秒

这个在日志、文件命名、数据记录里都很常见。

## 9.5 `strptime()`

和 `strftime()` 方向相反。

它是把“字符串形式的时间”解析成时间对象。

## 9.6 `perf_counter()`

这个很重要。

它不是拿来显示“现在几点”，而是拿来做：

**高精度计时**

比如你想比较两段代码谁更快：

1. 记录开始时间
2. 跑代码
3. 记录结束时间
4. 做差

### 你现在最该掌握到什么程度

这阶段重点掌握：

- `sleep()`
- `strftime()`
- `perf_counter()`

其他先有印象就够。

## 十、文本进度条：你看到的不是“炫技”，而是字符串 + 循环 + `\r`

PPT 里有个文本进度条例子，这东西其实挺适合你现在学。

因为它把好几个知识点串起来了：

- `for`
- 字符串拼接
- `print(end="")`
- `\r`
- `time.sleep()`
- 格式化输出

### 它的核心原理

不是“打印很多行”，而是：

1. 每次都打印一行新内容
2. 但不换行
3. 用 `\r` 把光标拉回开头
4. 下一次输出把上一行覆盖掉

所以用户看起来像“同一行在动”。

### 你应该从这个例子学什么

不是学会背进度条代码，而是学会：

- 一个效果是怎么由多个基础知识拼出来的

这才是你后面写项目最需要的能力。
